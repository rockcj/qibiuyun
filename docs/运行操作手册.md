## 2. 快速开始

```bash
# 1. 克隆项目
git clone <repo-url>
cd qibiuyun

# 2. 创建虚拟环境（后端）
python -m venv venv
source venv/Scripts/activate   # Windows
# 或 source venv/bin/activate  # macOS/Linux

# 3. 安装后端依赖
cd backend
pip install -r requirements.txt

# 4. 安装前端依赖
cd ../frontend
npm install

# 5. 配置环境变量（见第 3 节）

# 6. 启动后端
cd ../backend
python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload

# 7. 新开终端，启动前端
cd frontend
npm run dev

# 8. 访问
# 浏览器打开 http://localhost:3000
```

## 3. 环境变量配置

### 3.1 配置文件层级

```
qibiuyun/
├── .env.local          ← 根配置（最高优先级，含密钥，已 gitignore）
├── .env.example        ← 模板（可提交 Git）
├── .env.secrets.local  ← 密钥文件（仅本地，已 gitignore）
└── backend/
    ├── .env            ← 后端默认配置（无密钥时兜底）
    └── .env.example    ← 后端模板
```

**加载优先级:** `.env.local` > `backend/.env` > 代码默认值

### 3.2 必配变量

在项目根目录创建 `.env.local`：

```ini
# ── 运行环境 ──
APP_ENV=development          # development | demo | production

# ── 数据库 ──
# PostgreSQL（推荐）
DATABASE_URL=postgresql://offergpt:YourPassword@118.145.179.97:5432/offergpt
# SQLite（本地开发兜底，无需外部服务）
# DATABASE_URL=sqlite+aiosqlite:///./offergpt.db

# ── Redis 缓存 ──
REDIS_URL=redis://:YourPassword@118.145.179.97:6379/1

# ── LLM API ──
LLM_PROVIDER=deepseek
LLM_API_BASE_URL=https://api.deepseek.com/anthropic
LLM_MODEL=deepseekV4pro
LLM_API_KEY=sk-your-api-key-here
DEEPSEEK_API_KEY=sk-your-api-key-here

# ── 前端地址 ──
CORS_ALLOW_ORIGINS=http://localhost:3000
```

### 3.3 可选变量

```ini
# ── 语音服务 ──
ASR_PROVIDER=whisper           # 语音识别厂商
ASR_MODEL=whisper-1            # 识别模型
TTS_PROVIDER=edgeTts           # 语音合成厂商
TTS_VOICE=en-US-JennyNeural    # 合成音色
ENABLE_MOCK_ASR=true           # 开发模式：模拟语音识别
ENABLE_MOCK_TTS=true           # 开发模式：模拟语音合成

# ── 调度控制 ──
LOG_LEVEL=INFO                 # DEBUG | INFO | WARNING | ERROR
SESSION_TTL_SECONDS=7200       # 会话超时（秒）
REQUEST_TIMEOUT_SECONDS=30     # API 请求超时

# ── 后端服务 ──
BACKEND_HOST=0.0.0.0
BACKEND_PORT=8000
SESSION_TOKEN_SECRET=change-me-in-production

# ── 前端配置 ──
NEXT_PUBLIC_API_URL=http://localhost:8000
```

### 3.4 可切换的 DATABASE_URL

| 场景 | DATABASE_URL |
|------|--------------|
| 本地开发（纯 Python） | `sqlite+aiosqlite:///./offergpt.db` |
| 连接远程 PostgreSQL | `postgresql://user:pass@host:5432/dbname` |
| 本地 PostgreSQL | `postgresql://user:pass@localhost:5432/offergpt` |

> **提示:** `config.py` 中 `_resolve_url()` 会自动为 `postgresql://` 前面加上 `+asyncpg` 驱动标识。

## 5. 停止服务

### Windows

```bat
# 停止所有 Python（后端）
taskkill /F /IM python.exe

# 停止所有 Node（前端）
taskkill /F /IM node.exe

# 或按端口停止
netstat -ano | findstr :8000
taskkill /PID <PID> /F
```

### 清理端口占用

```bash
# 查找占用进程
netstat -ano | grep ':8000.*LISTEN'

# 强制释放
for pid in $(netstat -ano | grep ':8000' | awk '{print $5}' | sort -u); do
  taskkill //PID $pid //F
done
```

## 6. 数据库操作

### 6.1 建表

启动后端时自动执行，无需手动操作。

```python
# 等价于：
from database import init_db
import asyncio
asyncio.run(init_db())
```

### 6.2 查看表结构

```bash
# PostgreSQL 直连查看
python -c "
import asyncio, asyncpg
async def list_tables():
    conn = await asyncpg.connect(
        host='118.145.179.97', port=5432,
        user='offergpt', password='YourPassword', database='offergpt'
    )
    tables = await conn.fetch(
        \"SELECT tablename FROM pg_catalog.pg_tables WHERE schemaname='public'\"
    )
    for t in tables:
        print(t['tablename'])
    await conn.close()
asyncio.run(list_tables())
"
```

## 9. 常见问题

### Q1: 后端启动报 `Port 8000 already in use`

```bash
# Windows
netstat -ano | findstr :8000
taskkill /PID <PID> /F

# 然后重新启动
```

### Q2: PostgreSQL 连接报 `pg_hba.conf entry`

**原因:** 服务器未允许你的公网 IP。

**解决:** 联系管理员在 `pg_hba.conf` 添加你的 IP：

```
host    offergpt    offergpt    <你的IP>/32    scram-sha-256
```

**临时方案:** 把 `DATABASE_URL` 改为 SQLite。

### Q3: Redis 连接失败

- Redis 连接失败会自动降级为**内存缓存**，不影响核心功能
- 内存缓存的数据在进程重启后丢失
- 检查 `REDIS_URL` 格式：`redis://:password@host:port/db`

### Q4: 前端 404 或 API 调用失败

1. 确认后端正在运行：`curl http://localhost:8000/health`
2. 确认前端 `next.config.ts` 中 API 代理配置正确
3. 确认 `.env.local` 中 `NEXT_PUBLIC_API_URL=http://localhost:8000`

### Q5: npm install 慢或失败

```bash
# 清除缓存重试
npm cache clean --force
npm install

# 或使用国内镜像
npm install --registry=https://registry.npmmirror.com
```

### Q6: Python 依赖安装失败（asyncpg）

Windows 上 `asyncpg` 可能需要 Visual C++ Build Tools：

```bash
# 方案 1：安装构建工具
# 下载 Visual Studio Build Tools → 勾选 "C++ build tools"

# 方案 2：用 SQLite 开发（不需要 asyncpg）
# .env.local: DATABASE_URL=sqlite+aiosqlite:///./offergpt.db
```

### Q7: 前后端跨域报错

检查 `CORS_ALLOW_ORIGINS` 是否包含前端地址：

```ini
# 开发
CORS_ALLOW_ORIGINS=http://localhost:3000,http://127.0.0.1:3000

# 生产（改为实际域名）
CORS_ALLOW_ORIGINS=https://your-domain.com
```

## 10. Git 提交规范

### 分支策略

```
main          ← 主分支（可部署）
  ├── feat/xxx  ← 功能开发分支
  ├── fix/xxx   ← Bug 修复分支
  └── chore/xxx ← 杂项（文档、配置等）
```

### 提交消息格式

```
<type>: <简短描述>

<详细说明（可选）>

Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>
```

**Type:** `feat` | `fix` | `refactor` | `docs` | `chore` | `test`

### 示例

```
feat: 添加实时语音 WebSocket 通道

- 实现音频帧的接收和转发
```
