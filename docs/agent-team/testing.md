# AgentTeam 测试文档

## 文档目标

本文档面向测试 Agent，负责 SpeakUp AI 的功能验收、接口验证、实时链路稳定性、Demo 兜底路径和回归检查。测试重点不是追求完整企业级测试体系，而是保证三大场景稳定可用，新模块接入时不得破坏已有场景闭环。

## 测试优先级

| 优先级 | 场景模块 | 测试目标 | 通过标准 |
|---|---|---|---|
| P0 | 面试场景 | 验证简历/JD、Persona、实时面试、轻纠正、Offer Report、VAR 闭环 | 面试主流程可独立演示 |
| P1 | 餐厅点餐 | 验证点餐主题和服务员角色接入后不影响 P0 | 面试回归通过 |
| P2 | 商务会议 | 验证会议主题和会议角色接入后不影响 P0/P1 | 面试/点餐回归通过 |

## 测试范围

| 测试类型 | 目标 |
|---|---|
| 功能测试 | 验证三大场景端到端训练闭环 |
| 接口测试 | 验证 REST API 和 WebSocket 消息契约 |
| 实时链路测试 | 验证字幕、AI 回复、TTS 和轻纠正体验 |
| 报告测试 | 验证场景报告、分项评分和 VAR 事件 |
| 降级测试 | 验证 ASR、TTS、LLM 或麦克风异常时仍能演示 |
| 回归测试 | 验证每个 PR 合并后主分支保持可运行 |

## P0 核心验收用例

| 编号 | 场景 | 操作 | 期望结果 |
|---|---|---|---|
| TC-P0-001 | 首页入口 | 打开首页 | 能看到三大场景入口，均可进入 |
| TC-P0-002 | 面试配置 | 选择面试场景、Persona 和难度 | 成功进入面试资料配置流程 |
| TC-P0-003 | 简历解析 | 上传简历 | 返回 `resumeId` 和简历解析摘要 |
| TC-P0-004 | JD 解析 | 输入岗位 JD | 返回 `jobId` 和岗位解析摘要 |
| TC-P0-005 | 创建会话 | 使用 `resumeId`、`jobId` 创建 interview 会话 | 返回 `sessionId`、`sessionToken` 和 WebSocket 地址 |
| TC-P0-006 | 实时面试 | 点击开始并说一句英文回答 | 页面展示 ASR 字幕或 AI 面试官回复 |
| TC-P0-007 | 轻纠正 | 输入明显英语错误句子 | 页面展示一句话轻纠正 |
| TC-P0-008 | 连续追问 | 用户回答包含指标但缺少原因 | AI 面试官能追问具体细节 |
| TC-P0-009 | 结束会话 | 点击结束面试 | 进入 Offer Report 生成状态 |
| TC-P0-010 | 面试报告 | 打开报告详情 | 展示 Offer Score、分项分、建议和 VAR 事件 |

## 场景专项测试

### 面试场景

| 测试点 | 验收标准 |
|---|---|
| 简历解析 | 能抽取技能、项目和风险信号 |
| JD 解析 | 能抽取岗位技能、能力项和难度 |
| Persona | Founder、Product、Engineering 等模式至少能切换一种 |
| 连续追问 | 用户回答包含指标但缺少原因时，AI 能追问细节 |
| STAR 分析 | 行为题回答缺失 Result 时，VAR 或报告中能标记 |

### 餐厅点餐场景

| 测试点 | 验收标准 |
|---|---|
| 子主题 | 预约、点餐、投诉、结账至少支持一种完整流程 |
| 角色语气 | 服务员角色能体现友好或忙碌语气 |
| 功能句 | 报告能推荐点餐常用表达 |
| 礼貌表达 | 报告能展示礼貌用语评分或建议 |

### 商务会议场景

| 测试点 | 验收标准 |
|---|---|
| 项目汇报 | AI 能围绕进度、风险和下一步提问 |
| 礼貌打断 | 用户练习澄清或打断时，AI 能继续推进会议 |
| 时间控制 | 报告能指出表达过长或结构不清的问题 |

## 接口测试清单

| 接口 | 必测内容 |
|---|---|
| `GET /api/scenes` | 返回三大场景配置，各场景可通过开关控制 |
| `POST /api/resumes` | 支持上传文件并返回 `resumeId` |
| `POST /api/jobs` | 支持提交 JD 并返回 `jobId` |
| `POST /api/interviews` | 支持创建任意场景训练会话 |
| `GET /api/interviews/{sessionId}` | 能查询会话状态和配置 |
| `POST /api/interviews/{sessionId}/finish` | 能结束会话并触发报告 |
| `GET /api/interviews/{sessionId}/events` | 返回 VAR 时间轴 |
| `GET /api/interviews/{sessionId}/report` | 返回场景报告 |

## WebSocket 测试清单

| 消息 | 验收标准 |
|---|---|
| `audio.input` | 服务端能接收音频帧，不断开连接 |
| `asr.partial` | 前端能展示实时字幕 |
| `asr.final` | 前端能记录最终用户发言 |
| `agent.text.delta` | 前端能流式展示 AI 回复 |
| `tts.audio.delta` | 前端能播放或优雅降级到文本 |
| `timeline.event` | 前端能追加 VAR 事件 |
| `correction.light` | 前端能弹出轻纠正提示 |
| `control.finish` | 前端能进入报告流程 |

## 降级测试

| 异常 | 测试方法 | 期望结果 |
|---|---|---|
| 麦克风拒绝授权 | 浏览器拒绝权限 | 显示中文提示，可切换文本或 Demo |
| ASR 不可用 | 模拟 ASR 返回失败 | 用户可继续文本输入 |
| TTS 不可用 | 模拟 TTS 超时 | AI 文本仍正常显示 |
| LLM 超时 | 模拟模型超时 | 使用预置问题池回复 |
| 报告生成失败 | 模拟 Report Agent 报错 | 返回基础报告或重试提示 |
| WebSocket 断开 | 手动断网后恢复 | 前端尝试重连并展示状态 |

## PR 回归要求

每个 PR 合并前至少完成：

- 当前 PR 影响范围内的功能自测。
- 必须完成首页到场景报告的主流程验证。
- 修改共享模块时必须额外完成面试和点餐回归验证。
- `main` 分支运行方式确认。
- README 或 PR 描述中说明测试方式。
- 若引入第三方依赖，README 必须列明依赖和原创功能边界。

## 比赛前最终检查

- 三大场景入口均可打开并进入。
- 各场景可以完整完成语音或文本对话。
- 场景报告和 VAR 时间轴可以正常展示。
- 预置 Demo 数据可以在无麦克风情况下演示。
- 所有关键错误提示为中英文双语。
- 主分支代码可以重新安装依赖并启动。
