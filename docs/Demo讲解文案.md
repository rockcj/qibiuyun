# SpeakUp AI · Demo 讲解文案

> 七牛云 × XEngineer 暑期实训营 · 题目一：AI 英语口语陪练

---

## 开场（30 秒）

各位评委好，我是 SpeakUp AI 的开发者。

题目一要求做「AI 英语口语陪练」。市面上大多数方案是：选一个场景，语音转文字，AI 回复，朗读出来——四个步骤串起来就结束了。

但真正用过这类产品的人都知道，**能不能跑通和好不好用之间，差了一个工程深度**。

接下来我从三个维度展示 SpeakUp AI 的差异化。

---

## 第一部分：多场景不是换 Prompt，是完整的场景引擎（1 分钟）

我先打开首页。

（打开 http://localhost:3000 ，展示三个场景卡片）

三个场景：面试、点餐、会议。不是简单地换一段 System Prompt——每个场景有独立的：

- **评分维度**：面试 6 维（English/Logic/Confidence/STAR/Technical/Communication），点餐 5 维（含 Politeness），会议 5 维（含 MeetingControl）
- **AI 角色**：面试 5 种面试官人格，点餐 3 种服务员，会议 3 种角色
- **追问策略**：面试 5 条追问规则，点餐 6 条，会议 6 条
- **完整 Demo 兜底**：每个场景都有预置的完整会话数据

（点进面试场景 → 选 Founder 角色 → 创建会话）

面试场景还支持上传简历和 JD，系统用 LLM 提取技能画像，注入到 AI 面试官的上下文中。这是**个性化出题**，不是通用问答。

---

## 第二部分：实时语音管线的工程深度（1.5 分钟）

（进入对话页，展示语音对话）

这是整个系统最核心的部分。一条语音从麦克风到 AI 回复，经过 4 个环节：

```
VAD 检测 → Whisper ASR → DeepSeek LLM 流式 → EdgeTTS 合成
```

每个环节都可能失败。SpeakUp AI 的做法是**每一层都有降级**：

| 环节 | 正常路径 | 降级方案 |
|------|---------|---------|
| 输入 | 麦克风 + VAD | 自动切换文本输入 |
| ASR | 本地 Whisper | 8 层过滤器防误触发 |
| LLM | DeepSeek 流式对话 | 场景问题池兜底 |
| TTS | EdgeTTS 在线合成 | 浏览器 speechSynthesis |

（演示：拒绝麦克风权限 → 自动切文本模式 → 输入英文 → AI 回复）

VAD 这里还有一个细节：自适应静音阈值。短句子 0.5 秒停，长发言自动放宽到 1.2 秒——防止用户说长句时被误切断。

ASR 前有一层 **8 重过滤**：置信度、字数、重复检测、乱码过滤、中文检测……确保不会把环境噪音当作用户输入。

---

## 第三部分：纠错不打断——实时轻纠正（1 分钟）

（在对话中输入 "I have did a project last year"）

这是我认为最体现产品思考的地方。传统方案是：用户说错 → 弹出大段纠错 → 对话中断。体验很差。

SpeakUp AI 的做法是**异步轻纠正**：

1. 用户说完，AI 正常回复——**对话不中断**
2. 后台运行 Grammar Agent：12 条规则引擎优先，速度快；复杂情况调用 LLM 增强
3. 严重错误通过 `correction.light` 消息推送一个非模态 Toast

（展示 Toast 弹出 "Just a tip: we say 'have done'..."）

同时顶部实时显示 **Filler words 计数**（um/uh/like），用户可以关闭实时纠错开关。

---

## 第四部分：可量化的口语成长——场景报告（1 分钟）

（结束会话 → 跳转报告页）

训练的目的是进步。SpeakUp AI 的报告不是一段文字总结，而是**可量化的数据面板**：

- **场景专属评分**：面试 Offer Score 78，点餐 Score 82……每个维度有 evidence
- **SVG 雷达图**：纯前端绘制，无第三方依赖
- **VAR 时间轴**：每条事件带时间戳 + 音频片段，**可点击回放**
- **发音统计**：语速 WPM、停顿次数、低置信度词列表
- **语法纠正记录**：每条纠正关联到具体的 transcript 片段

（点击 VAR 时间轴事件 → 定位到对应对话轮次 → 播放音频）

## 第五部分：离线也能演示——三层降级 Demo（30 秒）

（打开 http://localhost:3000/demo）

比赛现场网络不稳定怎么办？SpeakUp AI 预置了**三层 Demo 降级**：

1. **API 层**：后端 /api/demo 返回最新数据
2. **localStorage 层**：浏览器缓存
3. **静态数据层**：前端内置完整数据，完全离线

（断网 → 刷新 → Demo 仍然完整展示）

三个场景各有独立 Demo 数据，评委任何时候都能看到完整效果。

## 收尾（30 秒）

总结一下 SpeakUp AI 的四个核心优势：

1. **场景引擎化**：不是换 Prompt，是完整的多维度场景配置系统
2. **管线健壮性**：VAD → ASR → LLM → TTS 全链路降级，每一层都有兜底
3. **纠错不打断**：异步 Grammar Agent + 非模态 Toast，对话流畅度不打折
4. **量化可回放**：场景报告不是 AI 总结，是带证据、带音频、可点击的复盘系统

谢谢评委。

## 备用：快速问答

**Q: 为什么用 WebSocket 而不是 WebRTC？**
A: 72 小时开发周期内，WebSocket 更容易调试和稳定。实际延迟在 500ms 以内，实时对话体验足够好。后期可以升级。

**Q: ASR 准确率怎么样？**
A: 本地 Whisper base 模型，配合 8 层 ASR 过滤器，实测误触发率低于 5%。支持运行时切换模型大小。

**Q: 如果 DeepSeek API 挂了怎么办？**
A: Demo 页面完全离线可用。实时对话有场景问题池兜底，不会白屏。

**Q: 前端的雷达图、时间轴用了什么库？**
A: 全部原创实现。RadarChart 是纯 SVG + CSS，TimelineViewer 是自建组件。前端零第三方 UI 库依赖。
