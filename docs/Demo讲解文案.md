## 1. 开场

各位评委好，我的项目叫 **SpeakUp AI**，一款 AI 实时场景英语口语实战教练。

题目一的核心是「帮用户在真实场景下练口语」。我做的不是一个对话机器人，而是一个**端到端的实时语音陪练系统**——从麦克风采集、语音识别、AI 对话生成，到语音合成、纠错反馈、课后量化报告，全部打通。

三个场景：求职面试、餐厅点餐、商务会议。下面我从场景引擎、实时管线、纠错策略、量化报告四个层面展开。

---

## 2. 场景引擎：不是换 Prompt，是完整的场景系统

（打开首页，展示三张场景卡片）

先说场景。很多人理解的多场景就是换一段 System Prompt——"你是一个面试官"改成"你是一个服务员"。但真实的口语陪练，不同场景需要的是完全不同的评估体系。

面试关心的是 STAR 结构、技术深度、沟通逻辑。点餐关心的是功能性表达、礼貌用语、任务完成率。会议关心的是项目汇报逻辑、打断技巧、会议控场。

所以 SpeakUp AI 把每个场景做成了**独立的配置引擎**：

- 面试 6 个评分维度，5 种面试官人格，上传简历和 JD 后 AI 会个性化出题
- 点餐 5 个评分维度，3 种服务员角色，预约、点餐、投诉、结账全流程
- 会议 5 个评分维度，3 种角色，汇报、提问、打断、总结都覆盖

（点进面试场景，选 Founder 角色，上传简历，创建会话）

每个场景的追问策略也是独立的——面试有面试的追问逻辑，点餐有点餐的引导方式。

## 3. 实时管线：每一层都有降级

（进入对话页）

这是系统最核心的部分。一条语音从用户说出来到 AI 回复，要走四个环节：

> 麦克风采集 → VAD 语音检测 → Whisper 语音识别 → DeepSeek 流式生成 → EdgeTTS 语音合成

每个环节都可能出问题。麦克风没权限、ASR 识别错了、LLM 超时了、TTS 服务挂了——线上产品这些都会遇到。

我的策略是**每一层都做降级**。不是坏了就报错，而是自动切换到备用方案，用户几乎无感知：

| 环节 | 正常 | 出了问题 |
|------|------|---------|
| 麦克风 | 实时采集 | 自动切成文本输入 |
| ASR | 本地 Whisper | 前面还有一层 8 重过滤，防误触发 |
| LLM | DeepSeek 流式 | 预置场景问题池兜底 |
| TTS | EdgeTTS | 浏览器内置语音朗读 |

（演示：拒绝麦克风权限，自动显示文本输入框，输入英文，AI 正常回复）

还有一个细节：VAD 的自适应静音检测。短句子 0.5 秒静音就判定结束，但如果你说了超过 2.5 秒，系统会认为你在组织长句，自动把静音阈值放宽到 1.2 秒——不会中途切断你。

ASR 前还有 8 层过滤器：置信度不够的过滤掉、重复的过滤掉、纯语气词的过滤掉、乱码的过滤掉——确保只有真正有效的语音才进入 LLM。

## 4. 纠错策略：不打断对话

（在文本框中输入 "I have did a project last year"）

这是我最花心思的地方——纠错的时机和方式。

传统做法是用户说错一句就弹一个对话框纠正，对话体验完全打断。我的思路是**异步轻纠正**：

1. 用户说完，AI 该怎么回怎么回，**对话不中断**
2. 后台异步跑 Grammar Agent，12 条规则引擎先扫一遍，复杂情况再用 LLM
3. 真有严重错误，通过 WebSocket 推一条 `correction.light` 消息，前端弹一个非模态 Toast

（TOAST 弹出 "Just a tip: we say 'have done' instead of 'have did'"）

这条提示一两秒就消失了，不影响对话。同时顶部实时显示 Filler Words 计数。如果用户觉得被打扰，随时可以关掉纠错开关。

## 5. 量化报告：不是一段 AI 总结

（结束会话，跳转报告页）

训练的目的是进步，进步需要量化。

大多数产品的报告是「请 AI 写一段总结」。我做的是一套**可量化的复盘面板**：

首先是场景评分：面试 78 分、点餐 82 分，每个维度都有评分 + evidence，告诉你哪里扣了分、依据是什么。

（展示雷达图）

然后是 VAR 时间轴——这是我最满意的设计。整个对话过程中，系统自动标记了发音问题、语法纠正、亮点表达，全部带上时间戳和音频片段。你点击任意一个事件，就能**直接跳到对应的对话位置**回放音频。

（点击 VAR 事件，定位到对话轮次）

还有个细节：发音统计里不只是 WPM 语速，还有停顿次数分布、ASR 低置信度词列表。语法纠正每一条都关联到了具体的对话片段——不是"你语法有问题"，而是"你在这个地方用错了"。

## 6. 离线兜底 + 收尾

（打开 /demo 页面）

最后一个想展示的——**离线 Demo**。比赛现场可能网络不稳，SpeakUp AI 设计了三层降级：

- 优先从后端 API 拿最新数据
- 后端连不上就用浏览器 localStorage 缓存
- 缓存也没有就用前端内置的静态数据

（断网，刷新 Demo 页，仍完整展示）

三个场景各有一套完整的预置数据——评分、雷达图、VAR 事件、对话回放——**断网也能完整演示**。

---

总结一下 SpeakUp AI 的四个设计理念：

**场景不是换 Prompt**，是独立的配置引擎、评分体系、追问策略。  
**管线不怕出问题**，VAD → ASR → LLM → TTS 每一层都有降级。  
**纠错不打断对话**，异步分析 + 非模态提示。  
**报告不是一段话**，是带证据、带音频、可点击回放的量化复盘。
